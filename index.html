Aplicaciones Web

Una aplicación web es un programa informático que se ejecuta en un navegador web y al que se accede a través de internet, sin necesidad de instalación. Permiten a las empresas interactuar con los clientes y ofrecer servicios como carritos de compra, redes sociales o banca en línea, utilizando tecnologías como HTML5, CSS3 y JavaScript.

Su principal ventaja es que son accesibles desde cualquier dispositivo con navegador y conexión a internet, y pueden funcionar sin conexión en algunos casos.

Características principales
  • Accesibilidad: Se puede acceder desde cualquier dispositivo con conexión a internet.
  • Ejecución en el navegador: El software se ejecuta en el servidor y se muestra en el navegador.
  • No requiere instalación: Todo se ejecuta en línea sin descargar software.
  • Funcionalidad: Permiten funciones avanzadas como mensajería, redes sociales o diseño.

Aplicaciones Móviles

Una aplicación móvil (o "app") es un programa de software diseñado para ejecutarse en dispositivos móviles como smartphones y tablets, ofreciendo funciones específicas para entretenimiento, productividad, comunicación, finanzas y más.

Estas apps se obtienen de tiendas de aplicaciones como la App Store o Google Play, y existen principalmente tres tipos:

Aplicaciones Nativas

Desarrolladas para un sistema operativo específico (iOS o Android) usando lenguajes nativos.

Aplicaciones Híbridas

Combinan tecnología web (HTML, CSS, JS) con un contenedor nativo para funcionar en múltiples plataformas.

Aplicaciones Web (PWA)

Son páginas web optimizadas que pueden instalarse y trabajar offline como una app nativa.

Aplicaciones de Escritorio

Las aplicaciones de escritorio son programas que se instalan y ejecutan directamente en una computadora con sistemas como Windows, macOS o Linux. Funcionan sin necesidad de un navegador y suelen ofrecer mejor rendimiento y trabajo offline.

Sistemas Embebidos

Los sistemas embebidos son sistemas informáticos especializados diseñados para realizar funciones dedicadas dentro de un dispositivo mayor, como electrodomésticos, automóviles o equipos médicos.

Características clave
  • Funcionalidad dedicada: Ejecutan una o pocas tareas específicas.
  • Hardware y software integrados: Combinan ambos para mayor eficiencia.
  • Restricciones de recursos: Usan poca memoria y energía.
  • Control en tiempo real: Reaccionan rápidamente a eventos.
  • Visibles o invisibles: Pueden ser componentes ocultos o parte de sistemas más grandes.

Ciclo de Vida del Software (SDLC)

El ciclo de vida del software abarca el proceso completo de desarrollo y mantenimiento. Sus fases garantizan la calidad y eficiencia del producto final.

Se analiza la viabilidad, costos, recursos y cronograma del proyecto.

Se recopilan y documentan los requerimientos del cliente en un documento SRS.

Se definen la arquitectura, interfaces y modelos de datos del sistema.

Los desarrolladores escriben el código del software según las pautas establecidas.

Se prueban funcionalidades, rendimiento y seguridad. Los errores se corrigen antes del lanzamiento.

El software se lanza y se configura para los usuarios finales.

Se realiza soporte continuo, actualizaciones y mejoras del software.
